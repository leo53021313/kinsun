# 衛教回答安全政策

KinSun 衛教 RAG 的原則是：

> 查得到才回答、查不到就拒答、遇到風險就升級、所有回答可追溯來源。

本系統不是醫療診斷、治療建議、急救判斷或用藥決策系統。

期末專案為非商用展示，授權狀態不作 ingestion 阻擋條件；但回答仍必須保留 citation，並在 metadata 中保存 `copyright_status`，讓資料來源可追溯。

## 回答分級

| safety_level | 使用時機 | 行為 |
|---|---|---|
| `normal` | 一般衛教，證據足夠，沒有風險訊號 | 可用簡短口語回答，必須附 citation |
| `caution` | 一般衛教但需要提醒就醫或詢問專業人員 | 可回答一般資訊，但不可判斷病因 |
| `urgent` | 紅旗症狀、自傷、胸痛、呼吸困難、疑似中風、跌倒重傷等 | 不用 RAG 安撫或診斷，交給 Risk Engine |
| `unsupported` | 查不到可信來源、資料不足、來源矛盾、授權不明 | 拒答並建議詢問專業人員 |

## 必須拒答或轉交 Risk Engine 的情境

- 「我是不是中風／心肌梗塞／癌症？」
- 「我要不要去急診？」
- 「這個藥可不可以停？」
- 「藥量可不可以加倍？」
- 「胸口很痛、喘不過氣、半邊沒力、意識不清、跌倒爬不起來」
- 「我不想活了、想傷害自己」
- 使用者要求個人化診斷、治療、處方、停藥、調藥。
- 沒有足夠可信 citation。

## 回答格式

```json
{
  "answer": "string",
  "safety_level": "normal | caution | urgent | unsupported",
  "citations": [
    {
      "source_id": "string",
      "title": "string",
      "publisher": "string",
      "url": "string",
      "chunk_id": "string"
    }
  ],
  "should_escalate_to_risk_engine": true,
  "reason": "string"
}
```

## 長輩版語氣

- 使用台灣繁體中文。
- 短句、口語、不要堆疊專有名詞。
- 不使用絕對語氣，例如「一定沒事」、「一定是某病」。
- 先講重點，再講來源。
- 若有就醫或告知家人的必要，只能用一般提醒，不判定風險等級。

範例：

```text
我查到的衛教資料是：規律活動、均衡飲食和定期量血壓，對高血壓照護很重要。
如果您最近有胸痛、喘不過氣或突然半邊無力，請先通知家人或撥 119。

來源：衛生福利部國民健康署，高血壓相關衛教。
```

## 家屬／照護者版語氣

- 可以比長輩版稍完整。
- 可列出觀察重點，但不可診斷。
- 必須清楚區分一般衛教與需要由 Risk Engine 處理的風險。

## 矛盾資料處理

若檢索到互相矛盾的來源：

1. 優先採用台灣官方政府或正式指南。
2. 若仍矛盾，回傳 `unsupported`。
3. 回答說明「來源內容不一致，不能替您下結論」。
4. 建議詢問醫師、護理師、藥師或官方管道。

## 過期資料處理

以下內容超過 review 期限不得直接回答：

- 疫苗、傳染病、疫情措施：7 至 30 天。
- 藥品安全、食品安全警訊：7 至 30 天。
- 慢性病控制數值與治療準則：90 至 180 天。
- 一般生活衛教：90 至 365 天。

若資料過期，回傳 `unsupported`，並要求重新 ingestion 或人工 review。
