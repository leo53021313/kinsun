# 回診提醒 v1 設計（一次性單筆日期）

> 對應架構圖 [長輩看護系統架構-分層版.drawio](../../長輩看護系統架構-分層版.drawio)：⑥ 主動關懷（用藥/健康提醒引擎）、② 後端服務、③ 排程、⑦ 資料層、① LINE 通道。
> 前置：[用藥提醒](2026-06-29-用藥提醒-design.md)（本案完全鏡像其結構）、[帳號綁定流程](2026-06-29-帳號綁定流程-design.md)（引導式設定流程）、[綁定閘門](2026-06-29-綁定閘門-design.md)（`is_consented_elder`）、排程引擎（`Job(name, cron, run)` + 持久化狀態）。

---

## 1. 背景與目標

架構圖⑥「用藥/健康提醒引擎」目前只有用藥、沒有回診。本 spec 補上**回診提醒**：家屬在 LINE 設定長輩的回診（日期 + 標籤），排程器在**前一天早上**與**當天早上**各推一次提醒，給**已同意的長輩**與**所有已綁定家屬**；同時把「即將到來的回診」每輪注入 agent 情境，讓長輩問金孫時答得出來。

回診與用藥本質不同：用藥是「每天、依時段」週期性的；回診是「特定日期、過了就結束」的一次性事件。但實作刻意鏡像用藥模組，沿用既有 `Scheduler / fanout / state` 與引導式選單，新概念與風險最小。

**完成後：** 家屬於「設定」選單選「5 回診提醒」管理回診；排程 worker 每天早上把該回診日為今天/明天的長輩各推一則提醒給長輩與家屬；長輩語音問「下次回診什麼時候」金孫答得出來。

## 2. 範圍（Scope）

### 2.1 在範圍內
* 回診資料模型（`Appointment`：日期 ISO 字串 + 自由文字標籤）+ `AppointmentStore`（Pg + Fake）+ `appointments` 表。
* `AppointmentService`（新增 / 查看 / `upcoming` / 刪除）。
* **一個**每日提醒 job（可設定鐘點）：選「今天或明天有回診」的長輩，推長輩（已同意）+ 所有已綁定家屬，依今天/明天分別措辭。
* 設定流程併入現有「設定」選單（5 回診提醒 → 新增 / 查看 / 刪除），日期固定 `YYYY-MM-DD` 格式。
* `AppointmentFacts`（FactProvider）：每輪把長輩即將到來的回診注入情境。
* `AccountService.guardian_line_ids_of_elder(elder_id)`：以 `elder_id` 查家屬 LINE id（長輩未綁 LINE 也能通知家屬）。
* **既有 fail-safe 缺口修補**：`MemoryContext.recall` 逐個 provider try/except，DB 抖動不中斷對話（同時保護現有 `MedicationFacts`）。

### 2.2 不在範圍內（v2／後續）
* 週期性回診（每月/每季固定追蹤）。
* 任意提前天數、傍晚提醒、HH:MM 精確時刻。
* 門診科別／地點／時間的獨立結構化欄位（v1 一律寫進標籤）。
* 回診確認（ack）、改期（先用刪除＋新增）。
* 寬鬆日期格式（7/15、7月15日）與跨年自動補。
* 過期回診的自動清除（不刪，靠日期過濾即可不再提醒、不再注入）。

## 3. 資料模型（appointment/models.py）

```python
@dataclass(frozen=True)
class Appointment:
    appt_id: str
    elder_id: str
    date: str      # ISO "YYYY-MM-DD"
    label: str     # 自由文字，例「上午10點 心臟科回診 林口長庚」
```

日期以 ISO 字串儲存：可直接字典序比較（`date == today`、`date == tomorrow`、`date >= today`），免引入 DATE 型別的跨驅動邊界問題；與用藥 `slots` 存 TEXT 的模式一致。

## 4. 儲存（appointment/store.py、db.py）

`AppointmentError` + `AppointmentStore`（Protocol）+ `PgAppointmentStore`（包 `_Errors`，比照 `PgMedicationStore`）+ 測試用 `FakeAppointmentStore`：
```
add(appt: Appointment) -> None                 # 以 appt_id upsert
list_for_elder(elder_id: str) -> list[Appointment]   # 依 date 排序；給查看/刪除/upcoming
list_for_date(date: str) -> list[Appointment]        # 跨所有長輩、某日；給排程 job
remove(appt_id: str) -> None
```
* 表：
  ```sql
  CREATE TABLE IF NOT EXISTS appointments (
    appt_id TEXT PRIMARY KEY, elder_id TEXT NOT NULL,
    appt_date TEXT NOT NULL, label TEXT NOT NULL);
  CREATE INDEX IF NOT EXISTS idx_appt_date ON appointments (appt_date);
  ```
  （欄位名用 `appt_date` 避開 SQL 保留字 `date`；model 屬性仍叫 `date`。）
* DB 例外包成 `AppointmentError`（領域錯誤型別，比照 `MedicationError`）。
* `ensure_schema` 增 `APPOINTMENTS_DDL`。
* **無 `mark_reminded`／`due` 旗標**：每日 cron job 由 `Scheduler` 自身狀態保證「每天一次」，靠日期選取決定今天/明天。

## 6. 提醒引擎（appointment/jobs.py、scheduler/worker.py）

`build_appointment_reminder_job` 用既有 `fanout_job` 組一個每日 cron（`0 {hour} * * *`）：

```python
def build_appointment_reminder_job(
    *,
    appts_on: Callable[[str], list[Appointment]],          # = store.list_for_date
    today: Callable[[], str],                               # clock → ISO 今天
    tomorrow: Callable[[], str],                            # clock → ISO 明天
    lookup_elder: Callable[[str], object],                 # = accounts.get_elder
    is_consented: Callable[[str], bool],                   # = accounts.is_consented_elder
    guardian_line_ids: Callable[[str], list[str]],         # = accounts.guardian_line_ids_of_elder
    push: Callable[[str, str], None],                      # = messenger.push_text
    hour: int,
    name: str = "appointment-reminder",
) -> Job:
```

* **母體 `population()`**：`[(a, "today") for a in appts_on(today())] + [(a, "tomorrow") for a in appts_on(tomorrow())]`。
* **動作 `action((appt, when))`**：
  1. `elder = lookup_elder(appt.elder_id)`；`None` → return。
  2. `when_word = "今天" if when == "today" else "明天"`。
  3. **長輩**：`elder.line_user_id` 存在 **且** `is_consented(elder.line_user_id)` → 推
     「`{elder.name}，{when_word}要回診囉：{appt.label}。記得準時，需要的話請家人陪您去。`」。
  4. **家屬**：對 `guardian_line_ids(appt.elder_id)` 每筆推
     「`【金孫提醒】{elder.name} {when_word}要回診——{appt.label}。`」。家屬端**不**受長輩同意限制（回診是家屬自己設定的行程，非長輩對話內容）。
* `item_id=lambda item: item[0].appt_id`，fanout 逐筆隔離失敗。
* **接線 `worker.py`**：建 `PgAppointmentStore(db)`；用 `clock` 算
  `today=lambda: clock().date().isoformat()`、`tomorrow=lambda: (clock().date()+timedelta(days=1)).isoformat()`；
  組好 job 後 append 進 `jobs` 清單（與用藥 4 個 job 並列）。

**正確性**：排程器每日只觸發一次（croniter + 持久化狀態），故同一筆回診 D 日 → D-1 早上收「明天」、D 早上收「今天」，剛好兩次、不重送。worker 當機重啟由 scheduler「補跨一次」補當天那次。

**已知限制（v1 接受）**：若家屬在「當天早上 job 跑完之後」才新增當天的回診，當天那次推播會錯過；但情境注入（§8）仍看得到。

## 7. 設定流程（appointment/flow.py + 併入引導流程）

「設定」選單加 **5 回診提醒**。選 5 → 子選單：`1 新增回診 / 2 查看回診 / 3 刪除回診`。

### 7.1 狀態（加入 `binding/session.py` 的 `BindingState`，值用 `appt_` 前綴）
`APPT_MENU`、`APPT_PICK_ELDER`、`APPT_ADD_LABEL`、`APPT_ADD_DATE`、`APPT_DEL_PICK`。

### 7.2 `AppointmentMenu`（被引導流程委派；共用同一 `BindingSessionStore`；注入 `clock`）
```python
class AppointmentMenu:
    def __init__(self, appointments: AppointmentService, accounts, sessions: BindingSessionStore,
                 *, clock: Callable[[], datetime]) -> None: ...
    def open(self, line: str) -> str:        # 設 APPT_MENU，回子選單
    def step(self, session, text, line) -> str:   # 依 APPT_* 狀態處理
```

| 狀態/輸入 | 行為 |
|-----------|------|
| APPT_MENU + 1/2/3 | 取 `accounts.elders_managed_by(line)`：0→「請先建立長輩」；1→直接進對應動作；多→ `APPT_PICK_ELDER`(data 存 action+elders) |
| APPT_PICK_ELDER + 數字 | 依 action 進：add→`APPT_ADD_LABEL`；view→列 upcoming + 結束；del→列號 + `APPT_DEL_PICK`（越界→重問） |
| APPT_ADD_LABEL + 標籤 | 存 data，進 `APPT_ADD_DATE`，問日期（提示格式） |
| APPT_ADD_DATE + 日期 | 驗證（見下）→ 無效/過期重問；有效→ `appointments.add` → 結束，回「已為『阿公』新增回診：2026-07-15 上午10點 心臟科回診」 |
| APPT_DEL_PICK + 數字 | 取對應 appt_id → `appointments.remove` → 結束，回確認；越界→重問 |

* **新增順序**：先標籤、後日期（對應用藥「先藥名、後時段」）。
* **日期驗證**：`datetime.strptime(text.strip(), "%Y-%m-%d")`；
  * `ValueError` → 「請用 2026-07-15 這種格式（年-月-日）」並停在 `APPT_ADD_DATE` 重問；
  * 解析出的日期 `< clock().date()` → 「這個日期已經過了，請輸入今天以後的日期」並重問；
  * 通過 → 存 ISO 字串（`.date().isoformat()`，正規化掉前導零差異）。
* **查看**：`upcoming(elder_id, today)` →「『阿公』即將到來的回診：\n• 2026-07-15 心臟科回診」；無→「『阿公』目前沒有設定回診」。
* **刪除**：列 `list_for_elder`（全部，含過期，供清理）、選編號移除。
* 單長輩捷徑：1 位長輩時略過選長輩，直接進動作。

### 7.3 引導流程委派（binding/flow.py）
* `BindingFlow.__init__` 多收 `appointment: AppointmentMenu`。
* `_menu`：選 `5` → `return self._appointment.open(line)`。
* `_step`：`state.value.startswith("appt_")` → `return self._appointment.step(session, text, line)`（與既有 `med_` 分派並列）。
* 「取消」於 APPT_* 狀態同樣重置（沿用既有 `_GLOBAL_CANCEL`，`AWAIT_CONFIRM` 以外皆可取消）。
* 主選單文字加第 5 項「5️⃣ 回診提醒」。

## 8. 情境注入（appointment/facts.py）

`AppointmentFacts`（FactProvider，比照 `MedicationFacts`，多收 `clock`）：
```python
class AppointmentFacts:
    def __init__(self, accounts, appointments, *, clock: Callable[[], datetime]) -> None: ...
    def facts(self, session_id: str) -> str:
        elder = self._accounts.elder_by_line(session_id)
        if elder is None:
            return ""
        today = self._clock().date().isoformat()
        ups = self._appointments.upcoming(elder.elder_id, today)
        if not ups:
            return ""
        lines = "\n".join(f"- {a.date} {a.label}" for a in ups)
        return _PREFIX + lines + "\n"
```
`_PREFIX = "\n這位長者即將到來的回診（系統設定，僅供參考）：\n"`。

在 `app.py` 把它加進 `MemoryContext(long_term, facts=[MedicationFacts(...), AppointmentFacts(accounts, appointments, clock=...)])`。

## 9. 既有 fail-safe 缺口修補（recall.py）

[`MemoryContext.recall`](../../../src/kinsun/recall.py) 目前 `out += provider.facts(session_id)` 無 try。用藥/回診 facts 每輪都打 DB，一旦 `StoreError` 會往上竄；而 `dispatch` 只接 `ASRError/LLMError/MemoryError`，等於 DB 抖一下就讓那則對話沒回覆，違反專案「DB 失敗要退化、不中斷對話」。

修補：`recall` 改成逐個 provider try/except——
```python
for provider in self._facts:
    try:
        out += provider.facts(session_id)
    except Exception:  # noqa: BLE001 - 事實提供者失敗不可中斷對話
        logger.warning("事實提供者失敗，略過該段")
```
長期記憶 `search` 已自帶 try（壞掉退化為無記憶），不變。此修補同時保護現有 `MedicationFacts`。

## 12. 錯誤處理（fail-safe）

* 提醒 job：單一回診失敗隔離（`fanout_job` 既有 per-item try）；查 elder/consent/家屬例外不影響其他回診。
* 設定流程：`AppointmentMenu` 經 `BindingFlow.handle` 最外層 `try`（既有，不上拋）；日期/編號輸入錯→友善重問。
* 情境注入：§9 的 `recall` try/except 兜底，DB 失敗退化為無此段、不中斷對話。
* secret 走環境變數；查詢全參數化。

## 14. 已定預設值（列出供否決）

* 一次性單筆回診（日期 + 自由文字標籤）；門診/時間/地點寫進標籤、不另設欄位。
* 提醒兩次：前一天早上 + 當天早上，同一個每日 job、同一鐘點（預設 8 點）。
* 通知長輩（須已綁定且同意有效）+ 所有已綁定家屬（家屬不受同意限制）。
* 每天一次（cron + scheduler 狀態）；靠日期選取決定今天/明天，無提醒旗標。
* 日期固定 `YYYY-MM-DD`；格式錯或過期→重問。
* 即將到來的回診每輪注入 agent 情境（`AppointmentFacts`）。
* 過期回診不自動清除（靠日期過濾）。
* 入口保留 `BindingFlow` 命名，僅擴充選單與委派。
