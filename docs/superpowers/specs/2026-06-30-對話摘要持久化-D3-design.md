# 對話摘要持久化設計（D 第三切片 / D3）

> 對應架構圖 [長輩看護系統架構-分層版.drawio](../../長輩看護系統架構-分層版.drawio)：⑦ 記憶/資料層、② 後端（報告資料源）。
> 前置：[長期記憶骨幹](2026-06-29-長期記憶骨幹-design.md)（`run_consolidation` 夜間每日整理）、[危急事件持久化 D1](2026-06-30-危急事件持久化-D1-design.md)、[提醒紀錄持久化 D2](2026-06-30-提醒紀錄持久化-D2-design.md)。
> 本案是「D 報告資料收集」的第三條（最後一條）資料流：對話摘要。之後 E 報告檢視讀取 D1+D2+D3。

---

## 1. 背景與目標

對話報告需要「長輩這天過得如何」的摘要，但目前只有逐輪 `turns`（每日整理後不完整）與 Mem0 事實（非每日摘要）。本案在既有夜間每日整理流程，用 LLM 把前一天對話摘成一兩句、落庫，作為對話報告（E）的資料源。

**完成後：** 每晚整理時，對有對話的長輩，用 LLM 產一段當日摘要並 upsert 到 `conversation_summaries`；摘要（LLM）失敗只 log、不影響長期記憶整理與其他長輩。

**本案不含**：對話報告 REST/前端與其權限把關（E）。

## 3. 資料模型與儲存（reports/summaries.py、db.py）

```python
@dataclass(frozen=True)
class ConversationSummary:
    session_id: str
    date: str          # ISO YYYY-MM-DD（被摘要的那一天，即前一天）
    content: str
    created_at: float


class ConversationSummaryError(Exception):
    """對話摘要讀寫失敗。"""


class ConversationSummaryStore(Protocol):
    def upsert(self, session_id: str, date: str, content: str) -> None: ...
    def list_for_session(self, session_id: str) -> list[ConversationSummary]: ...


class PgConversationSummaryStore:
    def __init__(self, db: Database, *, clock: Callable[[], datetime]):
        self._db = _Errors(db, lambda m: ConversationSummaryError(f"對話摘要存取失敗：{m}"))
        self._clock = clock

    def upsert(self, session_id, date, content) -> None:
        # INSERT(session_id, date, content, created_at=clock().timestamp())
        # ON CONFLICT (session_id, date) DO UPDATE SET content=EXCLUDED.content,
        #   created_at=EXCLUDED.created_at

    def list_for_session(self, session_id) -> list[ConversationSummary]:
        # SELECT ... WHERE session_id=%s ORDER BY date DESC
```

* DDL：
  ```sql
  CREATE TABLE IF NOT EXISTS conversation_summaries (
    session_id TEXT NOT NULL, date TEXT NOT NULL,
    content TEXT NOT NULL, created_at DOUBLE PRECISION NOT NULL,
    PRIMARY KEY (session_id, date));
  ```
* 複合主鍵 `(session_id, date)`：每長輩每日一筆、upsert（補跑同日不重複、覆蓋為最新）；PK 已含 `session_id` 前綴，免另設索引。
* `ensure_schema` 增 `CONVERSATION_SUMMARIES_DDL`。

## 4. 摘要產生（reports/summaries.py）

```python
SUMMARY_PROMPT = (
    "你是協助家屬了解長輩近況的助手。請把以下長輩與『金孫』的對話，"
    "用一兩句溫暖、客觀的台灣繁體中文摘要長輩這天的狀況與情緒。"
    "只根據對話內容，不要編造未提及的事。"
)


def summarize_day(session_id, *, short_term, summarizer, summaries, clock) -> None:
    turns = short_term.previous_day(session_id)
    if not turns:
        return
    content = summarizer.generate(system_prompt=SUMMARY_PROMPT, messages=turns)
    day = (clock().date() - timedelta(days=1)).isoformat()
    summaries.upsert(session_id, day, content)
```

* `summarizer` 即既有 `GeminiClient`（`.generate(system_prompt, messages) -> str`）；`short_term` 即 `PgMemoryStore`（`previous_day`）。
* 無前一天對話 → 不產摘要。
* `date` = 前一天（凌晨整理時 `clock().date()` 為當日，摘要的是 `-1` 天），與 `previous_day` 對齊。

---

## 5. 接線（scheduler/worker.py）

* 建 `summaries = PgConversationSummaryStore(db, clock=clock)`；加 module logger。
* `run_one`（每日整理 fanout 動作）：
  ```python
  def run_one(session_id):
      run_consolidation(session_id, short_term=memory, long_term=long_term)
      try:
          summarize_day(session_id, short_term=memory, summarizer=gemini,
                        summaries=summaries, clock=clock)
      except Exception:  # noqa: BLE001 - 摘要失敗不影響整理與其他長輩
          logger.warning("對話摘要失敗 session=%s", session_id)
  ```

## 6. 測試策略

**`tests/test_summaries.py`（離線）**：
* `summarize_day`（stub short_term 回固定 turns + stub summarizer 回固定字串 + `FakeConversationSummaryStore` + 固定 clock）→ 產一筆、`date` 為前一天、`content` 為摘要器輸出。
* 無前一天對話（stub 回 `[]`）→ 不產摘要。
* `PgConversationSummaryStore` upsert/list/同日覆蓋（opt-in `KINSUN_IT=1`）：同 `(session, date)` upsert 兩次只剩最新、`list_for_session` 依 date desc。

**`tests/fakes.py`**：`FakeConversationSummaryStore`（dict 以 `(session_id, date)` 為鍵；`list_for_session` 依 date desc）。

---

## 7. 錯誤處理（fail-safe）

* 摘要（LLM）失敗只 log、不影響長期記憶整理與其他長輩（fanout 逐筆隔離 + `run_one` 內 try/except）。
* 查詢失敗 → `ConversationSummaryError`（日後 E 處理）。

---

## 8. 已定決策（列出供否決）

* 摘要＝LLM 一兩句文字（非活躍度數據）；指令要求客觀、不可編造。
* 掛在既有夜間每日整理流程（同母體、同 `previous_day` 對話）；不另立 job。
* 鍵 `(session_id, date)`、upsert；`date` 為被摘要的前一天。
* `summarizer` 用既有 `GeminiClient`，無新依賴。
* 摘要為對話衍生內容；E 檢視時應以「家屬管理該長輩 + `can_view_transcript`」把關（本片只產出落庫）。
