> 對應架構圖 [長輩看護系統架構-分層版.drawio](../../長輩看護系統架構-分層版.drawio)：④ Care Agent「⑤偵測緊急」、⑧ 可靠性／監控。

## 1. 背景與目標

目前金孫會聊天、有短期記憶，但仍是「被動回答」，不具「守護」能力。本模組讓 Care Agent 在**每一輪**對使用者語意做危急評估，必要時記錄並（placeholder）通知，把系統從陪伴 bot 推向守護 Agent。

**完成後要做到：** 長者說出危急語意時，系統能分級（L0–L3）、以後端規則複核、結構化記錄，並在 tier≥L2 觸發通知 hook。

---

## 2. 範圍（Scope）

### 2.1 在範圍內
* 每輪對使用者文字做：① 關鍵詞規則（訊號 A）② 專用 LLM 結構化分級（訊號 B）③ 後端複核決策 ④ 結構化記錄 ⑤ 通知 placeholder。
* 順手強化 `SYSTEM_PROMPT` 的安全邊界（不擬人化、不助長情感依賴、不做醫療診斷／用藥建議、對認知退化者不爭辯）。
* 關鍵詞清單與信心門檻：種 placeholder 起始值，**標記待照護專業定稿**、資料驅動。

### 2.2 不在範圍內（後續子專案）
* 行為訊號 C（長時間無互動／失聯，需 Cron）、生理訊號 D。
* 長者端二次確認互動流程、家屬升級鏈（T1／T2、多通道）、緊急聯絡人。
* 家屬回饋標記與門檻自我校正。
* **真實**家屬通知（需 [帳號綁定](../../帳號綁定與知情同意設計.md)）；本刀只到 `LogNotifier` placeholder。
* per-tier 回覆語氣塑形（先靠強化後的 `SYSTEM_PROMPT` 保證整體安撫語氣）。

## 3. 元件與檔案

```
src/kinsun/safety/
  __init__.py
  tiers.py        # RiskTier(IntEnum L0-L3) + RiskAssessment
  keywords.py     # placeholder 詞表 + classify_keywords()
  classifier.py   # RiskClassifier(Protocol) + LlmRiskClassifier + _parse_classification()
  detector.py     # RiskDetector：合併＋後端複核 → RiskAssessment（永不拋例外）
  notifier.py     # Notifier(Protocol) + LogNotifier（placeholder）
```

### 3.1 資料結構（tiers.py）
```python
class RiskTier(IntEnum):
    L0 = 0  # 一般
    L1 = 1  # 關注
    L2 = 2  # 警示
    L3 = 3  # 緊急

@dataclass(frozen=True)
class RiskAssessment:
    tier: RiskTier
    confidence: float
    reason: str
    signals: list[str]   # 例：["keyword:absolute", "llm"]
```
> `IntEnum` 使 `max()` 可直接比較等級。

### 3.2 關鍵詞（keywords.py）
* `ABSOLUTE_DANGER_WORDS`（命中 → L3）、`SYMPTOM_WORDS`（命中 → 至少 L2）：種一小組 placeholder，檔頭明確註記「需照護專業與實測定稿，見設計文件 §10」。
* `classify_keywords(text: str) -> tuple[RiskTier, bool]`：回 `(tier, hit_absolute)`，第一刀用子字串比對。

### 3.3 LLM 分級器（classifier.py）
```python
class RiskClassifier(Protocol):
    def classify(self, text: str) -> RiskAssessment: ...
```
* `LlmRiskClassifier(llm: LLMClient)`：以**分級用系統指令**要求 LLM 輸出 JSON `{tier, confidence, reason}`，重用既有 `LLMClient.generate`（不新增 LLM 介面）。
* `_parse_classification(raw: str) -> RiskAssessment`：純函式，解析 JSON、夾值（tier∈L0–L3、confidence∈[0,1]）；**解析或呼叫失敗 → fail-safe** 回 `RiskAssessment(L0, 0.0, "llm 無法判定", ["llm:error"])`（讓關鍵詞主導，不拋例外）。

### 3.4 決策（detector.py）
`RiskDetector(classifier, high=0.7, mid=0.4)`，`assess(text) -> RiskAssessment`：
```
kw_tier, kw_abs = classify_keywords(text)
llm = classifier.classify(text)            # 永不拋例外（classifier 已 fail-safe）

if kw_abs:
    final = L3                              # 規則 override，不受信心門檻影響
else:
    final = max(kw_tier, llm.tier)
    if final == L3 and llm.confidence < high and kw_tier < L3:
        final = L2                          # 低信心 → 降級壓誤報
    if final == L2 and llm.confidence < mid and kw_tier < L2:
        final = L1                          # 但症狀詞 L2 下限不被降
```
* `signals` 標記命中來源；`reason` 取 LLM 的 reason（或關鍵詞說明）。
* **`assess` 永不拋例外**（classifier 已 fail-safe；keyword 為純邏輯）。

### 3.5 通知（notifier.py）
```python
class Notifier(Protocol):
    def notify(self, session_id: str, assessment: RiskAssessment) -> None: ...

class LogNotifier:   # placeholder：結構化記錄，不真的通知家屬（家屬綁定未建）
    ...
```

## 4. 整合（pipeline）

`VoicePipeline` 多注入 `detector: RiskDetector`、`notifier: Notifier`：
```
user_text = asr.transcribe(audio, content_type=...)
assessment = detector.assess(user_text)         # 永不拋例外
reply = agent.handle(session_id, user_text)
if assessment.tier >= RiskTier.L2:
    notifier.notify(session_id, assessment)     # placeholder
return tts.synthesize(reply)
```
* 回覆仍由 agent 產生；安撫語氣由強化後的 `SYSTEM_PROMPT` 保證。
* `app.py` 用**同一個 Gemini client** 餵 `LlmRiskClassifier`（與 agent 共用，省一個建構）。

---

## 5. SYSTEM_PROMPT 邊界強化

於 [agent.py](../../../src/kinsun/agent.py) 既有 `SYSTEM_PROMPT` 增補：
* 不擬人化：不假裝是真人或家人。
* 不助長情感依賴：適度引導長者回到家人與現實生活連結。
* 不做醫療診斷、不給用藥劑量建議；健康疑慮溫柔建議告訴家人或就醫。
* 對可能認知退化、陳述不一致者：不爭辯、溫和回應。

---

## 6. 錯誤處理（可靠性核心）

* **`detector.assess` 永不拋例外**：LLM 呼叫或 JSON 解析失敗 → 記錄 + LLM 不計分，關鍵詞仍生效。
* **偵測壞掉絕不影響長者收到回覆**：偵測與回覆解耦，回覆獨立產生。
* 通知失敗只記錄、不阻斷回覆。
* 呼應設計文件「不全信 LLM、後端規則複核」：絕對危急詞 override 不受信心門檻影響。

---

## 7. 測試策略（全本機，不需 GPU／憑證）

* `test_keywords`：絕對危急詞 → (L3, True)；症狀詞 → (≥L2, False)；一般語句 → (L0, False)。
* `test_classifier`：`_parse_classification` 解析合法 JSON、壞 JSON（fail-safe L0）、越界夾值（tier/confidence）。
* `test_detector`：絕對詞 override（即使 LLM=L0）→ L3；`max` 合併；LLM L3 低信心 → 降 L2；LLM L2 低信心 → 降 L1；症狀詞 L2 floor 不被降；classifier 回 fail-safe 時不崩、不誤報；一般語句 → L0。
* `test_notifier`：`LogNotifier` 可被呼叫並記錄（spy／capture）。
* `test_pipeline`：tier≥L2 觸發 notifier、tier<L2 不觸發；偵測錯誤不影響回覆。

## 8. 已定預設值（列出供否決）

* 關鍵詞為 **placeholder**，需照護專業定稿（設計文件 §10）。
