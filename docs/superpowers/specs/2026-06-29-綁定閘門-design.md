---

## 1. 背景與目標

綁定流程已上線，長輩/家屬可在 LINE 自助綁定並記錄知情同意。但目前**任何人對金孫傳語音都會得到完整 AI 對話**（ASR→Agent→記憶→回覆），不論是否綁定、是否同意。這違反知情同意原則，也讓未綁定者無從被引導。

本 spec 加上**綁定閘門**：只有「已綁定且同意有效」的長輩，語音才走完整對話；其餘語音改回引導訊息，請對方先完成綁定。

**完成後：** 未同意者的語音不再進入 AI 對話路徑，而是收到綁定引導；撤回同意後語音對話自動停止。

---

## 2. 範圍（Scope）

### 2.1 在範圍內
* `AccountService.is_consented_elder(line_user_id) -> bool`：綁定且同意有效。
* `ConsentGate`（`binding/gate.py`）：包同意檢查，**fail-open**。
* webhook 語音路徑加閘門：未通過 → 回 `BIND_FIRST_PROMPT`，不進 pipeline。
* `app.py` 組裝注入。

### 2.2 不在範圍內（v2／後續）
* **擋文字**：文字維持走綁定流程（否則無法貼碼綁定）。
* 逐功能閘門、同意版本變更的重新簽署提示。
* 「沒綁過」vs「被撤回」差異化文案。
* 家屬端（非長輩）的語音情境特別處理。

---

## 3. 資料流

```
LINE 訊息 → webhook
  ├─ 語音 → ConsentGate.allows(session_id)?
  │     ├─ True  → pipeline.process（完整 AI，現狀）
  │     └─ False → reply_text(BIND_FIRST_PROMPT)（引導綁定，不進 pipeline）
  ├─ 文字 → binding.handle（不擋，現狀）
  └─ 其他（貼圖/圖片）→ NON_AUDIO_PROMPT（現狀）
```

`session_id` 即 `event.source.user_id`（長輩 LINE id）。閘門只關門語音；文字與其他維持現狀。

## 4. 元件與介面

### 4.1 同意檢查（accounts/service.py）
```python
def is_consented_elder(self, line_user_id: str) -> bool:
    elder = self._repo.get_elder_by_line(line_user_id)
    if elder is None:
        return False
    consent = self._repo.get_consent(elder.elder_id)
    return consent is not None and consent.revoked_at is None
```
唯讀；綁定且同意未撤回才 True。DB 失敗 → 拋 `AccountError`（由閘門 fail-open 吸收）。

### 4.2 閘門（binding/gate.py）
```python
class ConsentChecker(Protocol):
    def is_consented_elder(self, line_user_id: str) -> bool: ...

class ConsentGate:
    def __init__(self, checker: ConsentChecker) -> None: ...
    def allows(self, line_user_id: str) -> bool:
        try:
            return self._checker.is_consented_elder(line_user_id)
        except Exception:        # fail-open：DB 故障不鎖長輩
            logger.exception("同意檢查失敗，放行 line=%s", line_user_id)
            return True
```
`ConsentGate` 只依賴小介面 `ConsentChecker`（`AccountService` duck-typed 滿足），與 accounts 解耦、易測。

### 4.3 webhook（channels/line/webhook.py）
* `create_app(*, parser, pipeline, messenger, binding, gate)`；`_handle_events` 多收 `gate`。
* 語音分支於 `messenger.get_audio` 前：
```python
if not gate.allows(session_id):
    messenger.reply_text(reply_token, BIND_FIRST_PROMPT)
    continue
```
* 新常數 `BIND_FIRST_PROMPT = "金孫需要先完成綁定才能陪您聊天喔。請把家人給您的邀請碼貼到這裡，或回覆「設定」開始。"`

### 4.4 組裝（app.py）
沿用已建的 `accounts`，新增 `gate = ConsentGate(accounts)`，注入 `create_app`。

---

## 5. 錯誤處理（fail-safe）

* **同意檢查失敗**（DB 故障）→ `ConsentGate.allows` 回 `True`（fail-open），記 log；**不鎖住真長輩**，符合「絕不中斷對話」。
* 閘門通過後，pipeline 既有的 ASR/LLM/記憶 fail-safe 不變。
* secret 走環境變數；查詢全參數化。

---

## 6. 測試策略（單元全離線）

* `tests/test_accounts_service.py`：`is_consented_elder` —— 綁定+同意→True；未綁→False；撤回同意→False；無同意紀錄→False（用 `FakeAccountRepository`）。
* `tests/test_binding_gate.py`：`ConsentGate.allows` 委派 checker；checker 拋例外 → 回 True（fail-open）。
* `tests/test_webhook.py`：未同意語音 → 回 `BIND_FIRST_PROMPT` 且 pipeline 不被呼叫（spy pipeline / fake gate）；已同意語音 → pipeline 跑、回其輸出；文字與貼圖路徑不受影響。

---

## 7. 已定預設值（列出供否決）

* 只擋語音；文字維持走綁定流程（保留貼碼綁定能力）。
* 「可服務」＝綁定且同意 `revoked_at` 為 None。
* fail-open：同意檢查例外時放行。
* 單一引導文案，不分「沒綁過/被撤回」。
* 每則語音做一次同意檢查（一次 DB 讀，低流量不快取）。
