## 1. 背景與目標

短期記憶（今日逐字對話）已完成。長期記憶在架構上分三塊：**長期記憶骨幹（路由）／知識圖譜（結構化事實）／向量庫（episodic 情緒）**。本 spec 做**第一塊：知識圖譜的結構化事實**。

**完成後：** 每日把對話抽成穩定事實（用藥／慢性病／家庭／個資／事件），帶**來源可信度（provenance）**；回覆前撈出餵 agent，讓金孫「記得長者的長期狀況」，但**不把長者自述當成已確認的醫療事實**。

---

## 2. 範圍（Scope）

### 2.1 在範圍內
* 每日批次（**可觸發函式 + CLI**）：今日短期對話 → LLM 抽取結構化事實 → 存 SQLite。
* 每筆事實帶 `category`、`provenance`、`confidence`。
* 回覆前檢索該長者事實 → 注入 `SYSTEM_PROMPT`（標註自述/未確認）。
* 短期（`kinsun_memory.db`）與長期（`kinsun_knowledge.db`）**分開存**。

### 2.2 不在範圍內（後續）
* 向量庫（episodic 情緒，下一塊）、長期記憶骨幹路由（兩塊都有後）。
* Cron 排程（切片②；本 spec 批次先手動/CLI 觸發）。
* 真圖譜關聯查詢／因果（v1 先「結構化事實表」，非圖資料庫）。
* 線上 DB：`FactStore` 介面不變，未來換 `PgFactStore`（Postgres）。
* 事實的時間軸／更新合併（v1 去重即可，不做版本化）。

## 3. 元件與檔案

```
src/kinsun/knowledge/
  __init__.py
  facts.py        # Provenance / FactCategory enum + Fact dataclass
  store.py        # FactStore(Protocol) + SqliteFactStore（去重）
  extractor.py    # FactExtractor(llm) + _parse_facts（純函式、fail-safe）
  recall.py       # format_facts_for_prompt() + KnowledgeRecaller
  batch.py        # run_fact_extraction() + main()（CLI）
```

### 3.1 資料結構（facts.py）
```python
class Provenance(str, Enum):
    SELF_CLAIMED = "self_claimed"        # 本人自述
    INFERRED = "inferred"                # 推測
    FAMILY_CONFIRMED = "family_confirmed" # 家屬確認（綁定後才會用）

class FactCategory(str, Enum):
    PROFILE = "profile"        # 個資
    FAMILY = "family"          # 家庭
    MEDICATION = "medication"  # 用藥
    CONDITION = "condition"    # 慢性病
    EVENT = "event"            # 事件
    OTHER = "other"

@dataclass(frozen=True)
class Fact:
    session_id: str
    category: FactCategory
    content: str
    provenance: Provenance
    confidence: float
```
> v1 不放 `created_at`；插入順序（rowid）即足夠，需時間軸再加（YAGNI）。

### 3.2 儲存（store.py）
```python
class FactStore(Protocol):
    def add(self, fact: Fact) -> None: ...
    def all_for(self, session_id: str) -> list[Fact]: ...
```
`SqliteFactStore(db_path)`：
* 表 `facts(id INTEGER PK, session_id, category, content, provenance, confidence REAL)`，唯一索引 `(session_id, category, content)`。
* `add`：`INSERT OR IGNORE`（**去重**：同 session+類別+內容不重複，避免每日重抽爆量）。
* `all_for`：回該 session 全部事實，依 `id` 排序（插入序）。
* DB 例外包成 `KnowledgeError`。

### 3.3 抽取（extractor.py）
* `FactExtractor(llm: LLMClient)`：用抽取系統指令要求 LLM 輸出 **JSON 陣列** `[{category, content, provenance, confidence}, ...]`，重用既有 `LLMClient.generate`。
* `_parse_facts(session_id, raw) -> list[Fact]`：純函式；逐筆解析、未知 category→`OTHER`、未知 provenance→`INFERRED`、clamp confidence；**整段壞 JSON → 空 list**（fail-safe）；跳過缺 content 的項目。
* 抽取指令明示：**健康宣稱若僅長者自述、未經確認 → `provenance=self_claimed`，不可標為已確認**（落地「糖尿病可信度」）。
* LLM 呼叫失敗 → 空 list（不拋例外）。

### 3.4 檢索與注入（recall.py）
* `format_facts_for_prompt(facts) -> str`：無事實 → `""`；有則回一段前綴，明示「以下為長者**自述**、未必經確認，請勿當醫療診斷」，逐筆列 `[類別中文] 內容（來源中文）`。
* `KnowledgeRecaller(store: FactStore)`：`recall(session_id) -> str = format_facts_for_prompt(store.all_for(session_id))`。

### 3.5 批次（batch.py）
* `run_fact_extraction(session_id, *, short_term, extractor, store) -> int`：讀今日短期 → 抽取 → 逐筆 `add` → 回新增筆數。
* `main(argv)`：CLI（`python -m kinsun.knowledge.batch <session_id>`），用 `load_settings` 組真實元件。

## 4. 整合

* **CareAgent**：`__init__(llm, memory, recaller)`；`handle` 先 `recaller.recall(session_id)`，把結果併進 `SYSTEM_PROMPT` 再呼叫 LLM。空字串 → 行為照舊。
* **app.py**：組裝 `SqliteFactStore(settings.knowledge_db_path)` → `KnowledgeRecaller` → 注入 `CareAgent`。
* **config.py**：新增 `knowledge_db_path`（預設 `kinsun_knowledge.db`）；`.env.example` 補上。
* `pipeline`/`webhook` 測試的 `CareAgent` 建構補一個 null recaller。

---

## 5. 錯誤處理（可靠性）

* 抽取、解析、檢索、DB **全 fail-safe**：壞掉退化成「沒有長期記憶」，**絕不中斷對話**。
* `recall` 失敗（DB 例外）→ 記錄並回 `""`（agent 照常回覆）。
* LLM 抽取失敗 → 空 list；批次回 0。
* 自訂 `KnowledgeError` 供辨識。

## 6. 測試策略（全本機，不需 GPU／憑證／Postgres）

* `test_knowledge_facts`：enum 值、`Fact` 建構。
* `test_knowledge_store`：add + all_for；**去重**（同 session+類別+內容只存一筆）；不同 session 隔離；enum round-trip。
* `test_knowledge_extractor`：`_parse_facts` 解析 JSON 陣列；壞 JSON → `[]`；未知 category/provenance 回退；clamp confidence；缺 content 跳過；fake LLM 整合。
* `test_knowledge_recall`：有事實 → 含 provenance 標註的字串；無事實 → `""`。
* `test_agent`：recaller 注入 → 其文字出現在傳給 LLM 的 system prompt；空 recall → `SYSTEM_PROMPT` 不變。
* `test_knowledge_batch`：stub 短期 + fake extractor + in-mem store → 事實存入、回筆數。

## 7. 已定預設值（列出供否決）

* 事實去重鍵 = `(session_id, category, content)`；重複直接忽略（不更新 provenance/confidence）。
* 檢索 = 該 session 全部事實（N 小，先不分類/不分頁）。
* `knowledge_db_path` 預設 `kinsun_knowledge.db`（已被 `.gitignore` 的 `*.db` 涵蓋）。
* 抽取重用 `LLMClient.generate` + JSON 解析（不新增 LLM 介面）。
* 無 embeddings、無 Postgres（v1 純 SQLite + stdlib）。
