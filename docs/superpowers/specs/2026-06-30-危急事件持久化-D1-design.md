# 危急事件持久化設計（D 第一切片 / D1）

> 對應架構圖 [長輩看護系統架構-分層版.drawio](../../長輩看護系統架構-分層版.drawio)：⑦ 記憶/資料層、⑧ 可靠性、② 後端（報告資料源）。
> 前置：[危急偵測核心](2026-06-29-危急偵測核心-design.md)（`RiskAssessment`/`RiskTier`）、[危急真實家屬通知](2026-06-29-危急真實家屬通知-design.md)（pipeline tier≥L2 通知）。
> 本案是「D 報告資料收集」拆解後的**第一條資料流**：危急事件持久化。後續：D2 提醒紀錄、D3 對話摘要；之後 E 報告檢視讀取這些資料。

---

## 1. 背景與目標

報告（E）需要資料，但目前危急事件只在通知器寫 log、未落庫，過了就消失。本案把 **tier≥L2 的危急事件持久化**，作為日後「健康報告」的資料源。一上線資料即開始累積（即使 E 尚未做）。

**完成後：** 每當 pipeline 偵測到 tier≥L2 並通知家屬時，同時寫一筆 `risk_events`（長輩、等級、原因、時間）；落庫失敗只 log、不影響對話與通知。

**本案不含**：報告 REST 端點與前端（E）；提醒紀錄（D2）、對話摘要（D3）。

* 欄位最小：`tier`（存 int）、`reason`、`created_at`。不存 confidence/signals（家屬報告用不到）。
* DDL：
  ```sql

    event_id TEXT PRIMARY KEY, session_id TEXT NOT NULL,
    tier INTEGER NOT NULL, reason TEXT NOT NULL, created_at DOUBLE PRECISION NOT NULL);
  CREATE INDEX IF NOT EXISTS idx_risk_events_session_created
    ON risk_events (session_id, created_at);
  ```

## 4. pipeline 接點（pipeline.py）

* `VoicePipeline.__init__` 多收 `risk_events: RiskEventStore`。
* `process` 既有 `if assessment.tier >= RiskTier.L2:` 分支內，先落庫（fail-safe）再通知：
  ```python
  if assessment.tier >= RiskTier.L2:
      try:
          self._risk_events.record(session_id, assessment)
      except Exception:  # noqa: BLE001 - 落庫失敗不可中斷對話
          logger.warning("危急事件落庫失敗")
      self._notifier.notify(session_id, assessment)
  ```
* 新增 module logger `logging.getLogger("kinsun.pipeline")`。
* 記錄門檻＝通知門檻（tier≥L2），語意一致：「事件＝通知過家屬的危急」。

## 6. 測試策略

**`tests/test_pipeline.py`（既有，注入 `FakeRiskEventStore`）**：
* tier≥L2 → `record` 被呼叫一次、`notify` 仍被呼叫；
* tier<L2 → `record` 不被呼叫；
* fail-safe：`record` 拋例外 → `process` 仍正常回 TTS、`notify` 仍被呼叫；
* 既有 pipeline 測試補注入 `risk_events` 參數。

**`tests/test_risk_events.py`（opt-in `KINSUN_IT=1`，比照 `test_pg_*`）**：
* `PgRiskEventStore` record L2 + L3 → `list_for_session` 依 `created_at` desc 回兩筆、`tier` 以 `RiskTier` 正確還原。

**`tests/fakes.py`**：`FakeRiskEventStore`（`record` 記錄呼叫於 `recorded` 清單；`list_for_session` 回對應 `RiskEvent`）。

---

## 7. 錯誤處理（fail-safe）

* 落庫失敗只 log、續跑：對話不中斷、家屬通知照發（資料先求落地，可靠性不犧牲對話）。
* 查詢失敗 → `RiskEventError`（日後 E 處理）。
* secret/連線走環境變數；查詢參數化。

---

## 8. 已定決策（列出供否決）

* 切片＝危急事件持久化；提醒紀錄/對話摘要為後續資料流。
* 記錄門檻 tier≥L2（與通知一致）。
* 欄位最小：tier(int)/reason/created_at；不存 confidence/signals。
* 落庫點在 pipeline tier≥L2 分支、fail-safe（不耦合通知器）。
* 以 `session_id`（長輩 LINE id）為鍵；E 以 elder→line id 反查。
* 不做事件保留/清理策略（YAGNI）。
