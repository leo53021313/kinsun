# 用藥提醒 v1 設計（固定時段）

> 對應架構圖 [長輩看護系統架構-分層版.drawio](../../長輩看護系統架構-分層版.drawio)：② 後端服務、③ 排程、⑦ 資料層、① LINE 通道。
> 前置：[帳號綁定流程](2026-06-29-帳號綁定流程-design.md)（引導式設定流程）、[綁定閘門](2026-06-29-綁定閘門-design.md)（`is_consented_elder`）、排程引擎（`Job(name, cron, run)` + 持久化狀態）。

---

## 1. 背景與目標

帳號綁定與閘門已上線，但金孫還不能主動提醒長輩吃藥。本 spec 加上**用藥提醒**：家屬在 LINE 設定長輩的用藥（藥名 + 時段），排程器在固定時段推播提醒給**已同意**的長輩。

採**固定時段（早/午/晚/睡前）**，完美套用現有每日 cron 排程；不做任意 HH:MM（v2）。

**完成後：** 家屬於「設定」選單管理用藥；排程 worker 在各時段鐘點推「{長輩}，{時段}該吃藥囉：藥A、藥B」給已同意長輩。

---

## 2. 範圍（Scope）

### 2.1 在範圍內
* 用藥資料模型（`MedicationSlot` 早/午/晚/睡前、`Medication`）+ `MedicationStore`（Pg + Fake）+ `medications` 表。
* `MedicationService`（新增/查看/刪除）。
* 4 個每日時段提醒 job（可設定鐘點），只推已同意長輩、同長輩同時段合併一則。
* 設定流程併入現有「設定」選單（4 用藥提醒 → 新增/查看/刪除）。
* `AccountService.get_elder(elder_id)`（job 推播查長輩）。

### 2.2 不在範圍內（v2／後續）
* 吃藥確認（ack）與未吃升級家屬。
* 任意 HH:MM、每 N 小時、指定星期。
* 編輯既有藥（先用刪除＋新增）。
* 同藥不同時段的個別劑量/備註。

## 3. 資料模型（medication/models.py）

```python
class MedicationSlot(StrEnum):
    MORNING = "morning"   # 早上
    NOON = "noon"         # 中午
    EVENING = "evening"   # 晚上
    BEDTIME = "bedtime"   # 睡前

SLOT_ORDER = (MORNING, NOON, EVENING, BEDTIME)          # 顯示／輸入順序（數字 1–4）
SLOT_LABELS = {MORNING: "早上", NOON: "中午", EVENING: "晚上", BEDTIME: "睡前"}

@dataclass(frozen=True)
class Medication:
    med_id: str
    elder_id: str
    name: str
    slots: tuple[MedicationSlot, ...]   # 一種藥可多時段（如 早+晚）
```

`slots_label(slots) -> str`：依 `SLOT_ORDER` 排序、以「、」連接中文標籤（如「早上、晚上」）。

## 4. 儲存（medication/store.py、db.py）

`MedicationStore`（Protocol）+ `PgMedicationStore` + `FakeMedicationStore`：
```
add(med: Medication) -> None
list_for_elder(elder_id: str) -> list[Medication]
list_for_slot(slot: MedicationSlot) -> list[Medication]
remove(med_id: str) -> None
```
* 表 `medications(med_id TEXT PRIMARY KEY, elder_id TEXT NOT NULL, name TEXT NOT NULL, slots TEXT NOT NULL)`；`slots` 存逗號連接的 slot 值（如 `"morning,evening"`）。
* `list_for_slot`：`WHERE slots LIKE %s`（參數 `f"%{slot.value}%"`；4 個 slot 值互不為子字串，安全）。
* DB 例外包成 `AccountError`（沿用既有例外型別，避免新型別）。
* `ensure_schema` 增 `MEDICATIONS_DDL`。
* **無 `mark_reminded`／`due`**：每日 cron job 由 `Scheduler` 自身狀態保證「每時段每天一次」。

## 6. 提醒引擎（medication/jobs.py、scheduler/worker.py）

`build_medication_slot_job`：
```python
def build_medication_slot_job(
    *,
    slot: MedicationSlot,
    meds_at_slot: Callable[[], list[Medication]],   # = lambda: store.list_for_slot(slot)
    lookup_elder: Callable[[str], object],          # = accounts.get_elder
    is_consented: Callable[[str], bool],            # = accounts.is_consented_elder
    push: Callable[[str, str], None],               # = messenger.push_text
    hour: int,
    minute: int = 0,
    name: str,                                      # 如 "medication-morning"
) -> Job:
    cron = f"{minute} {hour} * * *"
    def run() -> None:
        by_elder: dict[str, list[str]] = {}
        for med in meds_at_slot():
            by_elder.setdefault(med.elder_id, []).append(med.name)
        for elder_id, names in by_elder.items():
            try:
                elder = lookup_elder(elder_id)
                if elder is None or not elder.line_user_id:
                    continue
                if not is_consented(elder.line_user_id):
                    continue
                push(elder.line_user_id, f"{elder.name}，{SLOT_LABELS[slot]}該吃藥囉：{'、'.join(names)}")
            except Exception:   # noqa: BLE001 - 單一長輩失敗不影響其他
                logger.exception("用藥提醒失敗 elder=%s", elder_id)
    return Job(name=name, cron=cron, run=run)
```
* **合併**：同長輩同時段多藥 → 一則訊息列出。
* **只推已同意**：呼應綁定閘門；未綁定/未同意/未綁 LINE → 跳過。
* **補跨**：worker 當機後重啟，scheduler「補跨一次」會補當天該時段一次。
* `worker.py` 建 `PgMedicationStore`、`AccountService(PgAccountRepository(...), clock=...)`，加入 4 個 job（早/午/晚/睡前，鐘點取 config），與既有 job 並列。

## 7. 設定流程（medication/flow.py + 併入引導流程）

「設定」選單加 **4 用藥提醒**。選 4 → 子選單：`1 新增用藥 / 2 查看用藥 / 3 刪除用藥`。

### 7.1 狀態（加入 `binding/session.py` 的 `BindingState`）
`MED_MENU`、`MED_PICK_ELDER`、`MED_ADD_NAME`、`MED_ADD_SLOTS`、`MED_DEL_PICK`。

### 7.2 `MedicationMenu`（被引導流程委派；共用同一 `BindingSessionStore`）
```python
class MedicationMenu:
    def __init__(self, medications: MedicationService, accounts, sessions: BindingSessionStore) -> None: ...
    def open(self, line: str) -> str:        # 設 MED_MENU，回子選單
    def step(self, session: BindingSession, text: str, line: str) -> str:   # 依 MED_* 狀態處理
```
| 狀態/輸入 | 行為 |
|-----------|------|
| MED_MENU + 1/2/3 | 取 `accounts.elders_managed_by(line)`：0→「請先建立長輩」；1→直接進對應動作；多→ `MED_PICK_ELDER`(data 存 action+elders) |
| MED_PICK_ELDER + 數字 | 依 action 進：add→`MED_ADD_NAME`；view→列藥+結束；del→列號+`MED_DEL_PICK`（越界→重問） |
| MED_ADD_NAME + 藥名 | 存 data，進 `MED_ADD_SLOTS`，問時段 |
| MED_ADD_SLOTS + 時段 | `_parse_slots`（見下）→ 無效重問；有效→ `medications.add` → 結束，回「已為『阿公』新增『降血壓藥』（早上、晚上）」 |
| MED_DEL_PICK + 數字 | 取對應 med_id → `medications.remove` → 結束，回確認；越界→重問 |

* 「新增」單長輩捷徑：1 位長輩時略過選長輩，直接問藥名。
* 查看輸出：「『阿公』的用藥：\n• 降血壓藥（早上、晚上）\n• 鈣片（睡前）」；無藥→「『阿公』目前沒有設定用藥」。
* `_parse_slots(text) -> tuple[MedicationSlot, ...] | None`：擷取文字中的數字 1–4（容忍空白/逗號/連寫），對應 `SLOT_ORDER`，去重且按序；無有效數字→`None`。

### 7.3 引導流程委派（binding/flow.py）
* `BindingFlow.__init__` 多收 `medication: MedicationMenu`。
* `_menu`：選 `4` → `return self._medication.open(line)`。
* `_step`：`state` 屬 MED_* → `return self._medication.step(session, text, line)`。
* 「取消」於 MED_* 狀態同樣重置（沿用既有 `_GLOBAL_CANCEL` 機制，AWAIT_CONFIRM 以外皆可取消）。
* 主選單文字加第 4 項。

> 命名註記：入口類別目前叫 `BindingFlow`，實為「設定助手」；本案保留原名（改名動 webhook/app/測試，非必要），僅擴充選單與委派。

## 10. 錯誤處理（fail-safe）

* 提醒 job：單一長輩失敗隔離（沿用 scheduler 既有 per-job try）；查 elder/consent 例外不影響其他長輩。
* 設定流程：`MedicationMenu` 經由 `BindingFlow.handle` 的最外層 `try`（既有，不上拋）；時段/編號輸入錯→友善重問。
* secret 走環境變數；查詢全參數化（含 `LIKE` 參數化）。

## 12. 已定預設值（列出供否決）

* 固定時段 早/午/晚/睡前；鐘點預設 8/12/18/21（可設定）。
* 一種藥可多時段（`slots` 集合）；同藥同時段不重複。
* 每時段每日一次（cron + scheduler 狀態）；同長輩同時段多藥合併一則。
* 只推已綁定且同意有效的長輩。
* 時段以數字 1–4 複選輸入；無效重問。
* 入口保留 `BindingFlow` 命名，僅擴充。
