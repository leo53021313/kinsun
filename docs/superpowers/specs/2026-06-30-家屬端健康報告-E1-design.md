## 1. 背景與目標

D1（危急事件）、D2（提醒紀錄）資料已在累積。本案讓家屬在 LIFF 檢視某長輩近 30 天的**健康報告**：危急事件時間軸 + 用藥/回診提醒紀錄，經 REST 讀取；授權沿用既有「家屬管理該長輩」。

**完成後：** 家屬在 LIFF 長輩清單點「健康報告」→ 看到近 30 天的危急事件與提醒紀錄兩區塊。

**本案不含**：對話報告（E2，讀 D3、需 `can_view_transcript`）。

## 3. 後端 REST 端點（web/api.py）

`create_api_router(*, verifier, accounts, medications, appointments, clock, risk_events: RiskEventStore, reminder_logs: ReminderLogStore) -> APIRouter`（新增末兩參數）。沿用 `current_guardian`、`assert_manages`。

**`GET /api/elders/{elder_id}/health-report`**
```python
@router.get("/elders/{elder_id}/health-report")
def health_report(elder_id: str, line_user_id: str = Depends(current_guardian)) -> dict:
    assert_manages(line_user_id, elder_id)
    cutoff = (clock() - timedelta(days=30)).timestamp()
    elder = accounts.get_elder(elder_id)
    session_id = elder.line_user_id if elder else None
    risks = (
        [e for e in risk_events.list_for_session(session_id) if e.created_at >= cutoff]
        if session_id
        else []
    )
    reminders = [r for r in reminder_logs.list_for_elder(elder_id) if r.created_at >= cutoff]
    return {
        "risk_events": [
            {"tier": int(e.tier), "reason": e.reason, "created_at": e.created_at} for e in risks
        ],
        "reminders": [
            {"kind": r.kind, "content": r.content, "created_at": r.created_at} for r in reminders
        ],
    }
```
* 授權/存在性失敗 → 404（`assert_manages`）；無/失效 token → 401。
* `risk_events` 以長輩 LINE id（`session_id`）為鍵；長輩未綁 LINE → 危急空、提醒仍回。
* 兩清單皆已依 `created_at` desc（store 保證）；`tier` 回 int；`created_at` 回 epoch float。

**既有 4 個 API 測試**（`test_api_elders/medications/appointments/accounts`）的 `create_api_router(...)` 補傳 `risk_events=FakeRiskEventStore()`、`reminder_logs=FakeReminderLogStore()`（報告端點未觸及）。

## 4. 前端（frontend/）

* **`api.ts`**：
  ```typescript
  export type RiskEventItem = { tier: number; reason: string; created_at: number };
  export type ReminderItem = { kind: string; content: string; created_at: number };
  export async function getHealthReport(elderId):
      Promise<{ risk_events: RiskEventItem[]; reminders: ReminderItem[] }>   // GET .../health-report
  ```
* **`report.ts`**（新）：
  - `tierLabel(tier)`：`2→警示、3→緊急`，其餘 `L{tier}`。
  - `kindLabel(kind)`：`medication→用藥、appointment→回診`，其餘原樣。
  - `formatTime(epoch)`：`new Date(epoch * 1000).toLocaleString("zh-TW")`。
* **`pages/HealthReportPage.tsx`**（`/elders/:elderId/health-report`）：載入 → 兩區塊：
  - 「危急事件」：每列 `{formatTime} · {tierLabel} · {reason}`；空 → 「近 30 天無危急事件」。
  - 「提醒紀錄」：每列 `{formatTime} · {kindLabel} · {content}`；空 → 「近 30 天無提醒紀錄」。
  - 頂部「← 返回長輩清單」；錯誤顯示友善訊息。
* **`EldersPage`**：每長輩連結加「健康報告」→ `/elders/:id/health-report`。
* **`App.tsx`**：加 `<Route path="/elders/:elderId/health-report" element={<HealthReportPage />} />`。

## 6. 測試策略

**後端** `tests/test_api_health_report.py`（離線、TestClient + 局部 stub）：
* 局部 stub store 回可控 `created_at` 的 `RiskEvent`/`ReminderLog`（因既有 fake 不讓設 `created_at`）：
  ```python
  class _RiskEvents:
      def __init__(self, events): self._events = events
      def list_for_session(self, session_id): return self._events
  class _Reminders:
      def __init__(self, logs): self._logs = logs
      def list_for_elder(self, elder_id): return self._logs
  ```
* `clock=lambda: NOW`；seed 一筆近期 + 一筆 40 天前。
* 案例：
  * 管理中長輩（已綁 LINE）→ 200、`risk_events`/`reminders` 各只含近 30 天那筆；
  * 40 天前那筆被濾掉；
  * 未管理 elder（verifier 回他人）→ 404；
  * 無 token → 401；
  * 長輩未綁 LINE（`elder.line_user_id is None`）→ `risk_events` 空、`reminders` 仍回。

**既有 4 個 API 測試**：補傳 `risk_events=FakeRiskEventStore()`、`reminder_logs=FakeReminderLogStore()`，維持綠。

**前端**：`npm run typecheck` + `build`。

---

## 7. 錯誤處理（fail-safe）

* 授權/存在性 → 404；無/失效 token → 401。
* 後端 store 例外 → 500（不影響 webhook 對話主流程）。
* 前端：401 → 提示重新登入；其他非 2xx → 友善錯誤訊息。

## 8. 已定決策（列出供否決）

* 切片＝健康報告（危急 + 提醒）；對話報告為 E2。
* 近 30 天（以 `created_at` 截止篩選）；不做可調區間/分頁。
* 單一合併端點 `GET /health-report` 回兩區塊。
* 授權沿用 `assert_manages`（家屬管理該長輩）；健康資料不需 `can_view_transcript`（那是 E2 對話報告）。
* 沿用做法 A：擴充 `create_api_router`（+2 store 參數），既有 4 測試補參數。
* `tier` 回 int、前端映射中文；`created_at` 回 epoch、前端格式化。
