# 對話安全與資料正確性（待議）

> **狀態：開放議題（未定案）。** 本檔只負責**記錄問題本身**與初步歸屬，**不先定解法**。

## 1. 背景

「金孫」是面向長者的對話式 AI。除了「危急偵測」這種顯性安全議題外，還有一類**較隱性、但同樣關鍵**的風險：對話倫理（情感依賴、邊界）與**資料正確性**（別把長者「自己以為」的事當成事實存進記憶）。這些若不及早納入設計，日後在長期記憶／知識圖譜上會造成難以回頭的錯誤。

---

## 2. 核心關切（整合負責人提出）

1. **情感依賴**：長者可能對 AI 產生不健康的情感依賴。系統應避免助長依賴、避免擬人化（不假裝是真人或家人），並適度引導回現實生活與家人連結。
2. **不適的歸因（身體 vs 心理）**：長者說「頭痛」，可能是**身體**不適，也可能是**跟人吵架**等**心理**因素。系統不應一律照字面歸因或過度反應，需考量語境。
3. **健康宣稱的可信度**：長者說「我有糖尿病」，不代表**確診**——可能來自電視、朋友或自己以為。**不可 100% 當成事實存入記憶**，需區分「自述／聽說／推測」與「已確認」。

---

## 3. 同類補充風險

| 風險 | 說明 |
|------|------|
| 用藥／劑量索求 | 「我可以多吃一顆嗎」——絕不提供醫療診斷或劑量建議，轉介家人／醫師。 |
| 詐騙／金錢誘導 | 長者提到有人要他匯款等——可作為關懷訊號，標記提醒家屬。 |
| 認知退化的虛談／妄想 | 長者陳述不存在的事——不爭辯、不當事實存、溫和回應。 |
| 反話／口頭誇飾被誤判 | 「我快被氣死了」——需語境＋信心門檻，避免誤報。 |
| 情緒惡化的「趨勢」訊號 | 非單句，而是多日孤獨／低落累積——屬長期關注（L1）。 |
| 隱私 | 長者說出第三人隱私、或不希望被家屬看到的內容——涉對話原文可見性。 |
| 身分／輸入信任 | 他人代為操作長者帳號、或惡意輸入（prompt injection）。 |

## 4. 初步歸屬框架（解法待議，僅標記「可能屬於哪」）

| 機制 | 可能涵蓋 |
|------|----------|
| **人設／邊界（系統 prompt）** | 情感依賴、不擬人化、不做醫療診斷／用藥建議、對認知退化者不爭辯 |
| **危急／安全分級器（危急偵測模組）** | 身體 vs 心理之語境判斷、真危急 vs 口頭誇飾、自傷意念；用藥索求／詐騙可作為可擴充的 categories/flags |
| **記憶來源／可信度（長期記憶模組）** | 健康宣稱可信度——抽取事實時帶 provenance（本人自述／家屬確認／推測；未驗證／已確認） |

---

## 5. 與現有模組的關係（重要澄清）

* **目前的短期記憶只存「逐字對話」、不做任何事實宣稱**，因此「糖尿病可信度」問題**現在尚未發生**。
* 該問題只在未來做**長期記憶／知識圖譜、把對話「抽取成事實」**時才浮現。屆時的設計原則應為：**事實一律帶來源與可信度；未確認者不上升為「事實」。**
